Этот корпоративный веб-сайт был создан с упором на доступность, обеспечивая бесшовное взаимодействие для всех пользователей, включая тех, кто полагается на вспомогательные технологии. Помимо базового соответствия WCAG, процесс разработки включал сценарии тестирования с реальными пользователями и оптимизацию для скринридеров, поддерживаемую специальными аудитами доступности. Результатом стал фронтенд, который не только соответствовал техническим стандартам, но и обеспечивал отполированный, беспрепятственный опыт.

## Технологии

Платформа была построена на Next.js с гибридным рендерингом (SSR/статический) для производительности, поддерживаемая модулями бэкенда Node.js и Drupal в качестве отдельного CMS. Инфраструктура полагалась на GitLab CI/CD для атомарных развертываний, в то время как доступность обеспечивалась через автоматизированное тестирование (Lighthouse, axe-core) и ручные аудиты. Система дизайна фронтенда была задокументирована в Storybook, обеспечивая согласованность UI между командами.

## Обязанности

Как ведущий фронтенд-разработчик, я отвечал за сквозную доставку платформы—от проектирования CI/CD пайплайнов до координации API-контрактов с командами бэкенда для бесшовной интеграции headless CMS. Помимо основной разработки, я внедрил инструменты для обеспечения стандартов качества кода и поддержания согласованности системы дизайна в различных средах. Роль также включала написание серверной логики в Next.js для оптимизации производительности фронтенда, наряду с управлением циклами выпуска и документацией для обеспечения плавных развертываний.

## Ключевые достижения

- Достигнуто соответствие WCAG 2.1 AA с тестированием реальными пользователями
- Внедрены автоматизированные аудиты доступности в CI/CD пайплайн
- Построена масштабируемая система дизайна, задокументированная в Storybook
- Оптимизирована производительность с гибридным SSR/статическим рендерингом
